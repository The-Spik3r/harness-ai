# Implementation Report — STORY-008: Multi-stage Docker packaging for single-port image

**Plan**: `.agents/plans/PRD-002-reflex-chat-ui/completed/STORY-008-docker-single-port-packaging.plan.md`
**Epic Branch**: `epic/PRD-002-reflex-chat-ui`
**Commit**: `75e5ad6`

## Summary

Replaced the single-stage `Dockerfile` with a two-stage build following Reflex's own documented pattern for "single port, no Node/Bun in the final image" (`reflex-dev/reflex`'s `docker-example/production-one-port` reference): a Python+Node/Bun builder stage runs `reflex export --frontend-only --no-zip` to produce a static frontend bundle, then a Python-only final stage installs Caddy as the single-port front door — Caddy serves the static bundle directly and reverse-proxies both Reflex's reserved routes (`/_event`, `/ping`, `/_upload`) and this repo's own routes (`/query`, `/audit`, `/stats`, `/health`) to an internal-only Reflex backend process (`--backend-only`, fixed at container-internal port 8001, never exposed). No Redis — this app is single-instance, so Reflex's default disk-based state manager is sufficient. Added `Caddyfile`, extended `docker-compose.yml` (build arg for `PORT`, absolute `DATABASE_URL`), and extended `.dockerignore`.

Live-validated end-to-end with a real `docker-compose up -d --build`: `/health`, `/`, `POST /query` (a real OpenRouter call succeeded), `GET /audit`, `GET /stats` all worked through the single exposed port; confirmed zero Node/Bun binaries anywhere in the built image; confirmed the full 109-test suite passes unmodified when run via `docker-compose run --rm harness-ai pytest tests/ -v`.

## Deviations from Plan

1. **`reflex init --no-agents` required (not anticipated in the plan)**: bare `reflex init` in the builder stage failed with `UnicodeDecodeError: 'utf-8' codec can't decode byte 0x97` while trying to merge the existing `chat_ui/AGENTS.md` (generated by STORY-002, apparently containing a non-UTF-8 em-dash byte from its original generation). Since the Docker build has no use for `AGENTS.md` regeneration, added `--no-agents` to the `reflex init` invocation — a one-flag fix, no other files touched.
2. **WORKDIR kept at `/app`, not `/app/chat_ui` (plan's Task 2 draft had it at `chat_ui/`)**: the plan's draft Dockerfile set the final `WORKDIR /app/chat_ui` before `CMD` for Reflex's benefit, but this silently broke `docker-compose run harness-ai pytest tests/ -v` (already documented in `README.md`) since `tests/` lives at `/app/tests`, not `/app/chat_ui/tests`. Fixed by keeping the image's default `WORKDIR` at `/app` and having the `CMD` itself `cd chat_ui && exec reflex run ...` — Reflex still gets the CWD it needs at runtime, while the documented test-running workflow is preserved. Caught by actually running Task 6's validation, not assumed.
3. Confirmed (not a deviation, just noting): `reflex export`'s output landed at `.web/build/client/`, exactly as the plan's Task 2 note anticipated needing to verify — no path adjustment was needed.
