# Plan: Mount existing FastAPI app into Reflex (single-port process)

## Summary

Wire `chat_ui/chat_ui/chat_ui.py` to pass the existing `app.main.app` FastAPI instance into Reflex's `api_transformer`, so Reflex mounts its own routes (`/_event`, `/_upload`, `/ping`, compiled frontend) onto the harness's existing app rather than creating a second ASGI app. Because `chat_ui/` is its own Reflex project directory (sibling to `app/`, not a subpackage of it), the repo root must be added to `sys.path` before `app.main` can be imported — this is the one non-obvious wrinkle in an otherwise small change. Add a small pytest assertion that diffs PRD-001's route table against Reflex's reserved-route list (Risk 2 in PRD Section 14), and validate manually that all four existing endpoints plus the chat frontend are reachable on one port in `reflex run --env prod` mode.

**Correction to the story's acceptance criteria**: the story text says `api_transformer` is "configured" in `chat_ui/rxconfig.py`. That is not how the current Reflex API works (confirmed against live docs/source for `reflex==0.9.6.post1`, the version pinned in this repo): `api_transformer` is a keyword argument to `rx.App(...)` in the app's entrypoint module, not a field on `rx.Config`. `rx.Config` only exposes `backend_port`, `frontend_port`, `api_url`, `deploy_url`, `cors_allowed_origins`, etc. — no `api_transformer`. This plan implements the mount in `chat_ui/chat_ui/chat_ui.py` (where `rx.App()` is already constructed) and leaves `rxconfig.py` untouched. The end result — one FastAPI instance serving both the harness routes and Reflex — is unchanged; only the file that wires it differs from the story's literal wording.

Similarly, the story/PRD's `REFLEX_ENV=prod` is not a real Reflex environment variable for this version; the documented mechanism is the CLI flag `reflex run --env prod` (env-var equivalent, if ever needed by Docker packaging in STORY-008, is `REFLEX_ENV_MODE=prod`). This story does not need to introduce any new env var — it only needs to verify single-port mode works via the CLI flag. `.env.example` and the Dockerfile stay out of scope (that's STORY-008 per PRD Phase 5).

## Tasks

### Task 1: Mount the existing FastAPI app via `api_transformer`

- **File**: `chat_ui/chat_ui/chat_ui.py`
- **Action**: UPDATE
- **Implement**:
  1. At the top of the file, before `import reflex as rx`, insert a `sys.path` fix so `app.main` (which lives at the repo root, a sibling of `chat_ui/`, not inside it) is importable regardless of the process's cwd:
     ```python
     import sys
     from pathlib import Path

     REPO_ROOT = Path(__file__).resolve().parents[2]
     if str(REPO_ROOT) not in sys.path:
         sys.path.insert(0, str(REPO_ROOT))
     ```
     (`chat_ui/chat_ui/chat_ui.py` → `parents[0]` = `chat_ui/chat_ui`, `parents[1]` = `chat_ui/`, `parents[2]` = repo root.)
  2. Import the existing app: `from app.main import app as fastapi_app`.
  3. Change `app = rx.App()` to `app = rx.App(api_transformer=fastapi_app)`.
  4. Leave the placeholder `index()` page and `app.add_page(index)` as-is — STORY-004 replaces this page with the real chat UI; this story only wires the mount.
- **Mirror**: `app/main.py:16` for how `app.main.app` is constructed (nothing here changes that file).
- **Validate**: `cd chat_ui && python -c "import chat_ui.chat_ui"` fails today (module not runnable standalone outside `reflex run`) — instead validate via Task 3's manual `reflex run` check. ### Task 2: Add the route-collision assertion (PRD Section 14 Risk 2)

- **File**: `tests/test_route_reservations.py`
- **Action**: CREATE
- **Implement**:
  ```python
  import os

  os.environ.setdefault("OPENROUTER_API_KEY", "test-key")
  os.environ.setdefault("ADMIN_TOKEN", "test-token")

  from app.main import app

  # Reserved by Reflex >=0.8 (chat_ui/ pins reflex==0.9.6.post1); see PRD-002 Section 9.
  REFLEX_RESERVED_ROUTES = {"/ping", "/_event", "/_upload"}


  def test_no_route_collides_with_reflex_reserved_routes():
      harness_routes = {route.path for route in app.routes}
      collisions = harness_routes & REFLEX_RESERVED_ROUTES
      assert not collisions, f"Harness routes collide with Reflex reserved routes: {collisions}"


  def test_expected_harness_routes_present():
      harness_routes = {route.path for route in app.routes}
      assert {"/query", "/audit", "/stats", "/health"} <= harness_routes
  ```
  This is a static check against `app.main.app`'s route table — it does not import or run Reflex, so it stays fast and has no new test dependency. It fails loudly (per PRD Section 14 Risk 2's mitigation) if a future route is ever added under a reserved name.
- **Mirror**: `tests/test_main.py:1-16` for the env-default + import style used throughout this test suite.
- **Validate**: `pytest tests/test_route_reservations.py -v` — both tests pass.
